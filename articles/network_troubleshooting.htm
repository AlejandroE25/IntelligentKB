General Network Troubleshooting

network, internet, connectivity, firewall, DNS, IP address, ping, traceroute, troubleshooting
[Internal Staff Note] Network status dashboard: status.network.university.edu. For reported outages affecting multiple users, open a P1 ticket and page the on-call Network engineer. Port 443 and 80 must be open for all university systems; ports 1194 (OpenVPN) and 443 (AnyConnect) required for VPN.

Overview

This article provides general steps for diagnosing and resolving network connectivity issues on campus and off campus. Try the steps in order, from simplest to most advanced.

Step 1: Basic Checks

  • Confirm the device's Wi-Fi or Ethernet is enabled and connected.
  • Restart the device's network adapter: on Windows use ipconfig /release then ipconfig /renew; on macOS toggle the network interface off and on in System Preferences.
  • Try a different website or service to determine whether the issue is isolated to one site.

Step 2: Router and Modem (Off-Campus Users)

  • Power-cycle your home router and modem: unplug them, wait 30 seconds, then plug them back in (modem first, then router).
  • Connect another device to the same network to determine if the issue is device-specific or network-wide.
  • If the issue is network-wide, contact your ISP.

Step 3: DNS Troubleshooting

  • Try accessing a site by IP address to rule out DNS issues (e.g., http://8.8.8.8).
  • Flush DNS cache: Windows: ipconfig /flushdns; macOS: sudo dscacheutil -flushcache.
  • Temporarily switch DNS servers to 8.8.8.8 (Google) or 1.1.1.1 (Cloudflare) to test.

Step 4: Firewall and Antivirus

  • Temporarily disable any third-party firewall or antivirus to check if it is blocking connectivity.
  • If disabling the firewall resolves the issue, add the relevant university hostnames as trusted sites rather than leaving the firewall off.
  • Check Windows Defender Firewall settings: Control Panel → Windows Defender Firewall → Allow an app through the firewall.

Step 5: Diagnostic Commands

  • Ping: ping university.edu — checks basic connectivity and latency.
  • Traceroute: tracert university.edu (Windows) or traceroute university.edu (macOS/Linux) — identifies where packets are being dropped.
  • NSLookup: nslookup university.edu — checks DNS resolution.

Checking for Known Outages

Before spending time troubleshooting, check the university network status page at status.network.university.edu to see if a known outage is affecting services.

Contact

If none of the above steps resolve the issue, contact the Help Desk at help@university.edu or call 555-0100, providing the output of your ping and traceroute commands.
